Information Entropy (H)

Entropy measures the **uncertainty** or unpredictability of a source. In Cyber Security, high entropy is essential for password hashing algorithms and encryption keys.

$$H(X) = - \sum_{i=1}^{n} p(x_i) \log_2 p(x_i)$$
  • Password Strength: Higher entropy passwords require exponential brute-force guesses.
  • Intrusion Detection: A sudden spike in network traffic entropy indicates abnormal activity such as a DDoS or malware propagation.

Probability Distributions for Security

1. Binomial Distribution (Discrete)

Models the number of successes in $n$ independent trials with constant success probability $p$.

$$P(X=k) = \binom{n}{k} p^k (1-p)^{n-k}$$

Cyber Example: If 100 employees receive a phishing email ($n=100$) and the click-rate is 10% ($p=0.1$), calculate the probability of exactly 5 people clicking.

2. Poisson Distribution (Discrete)

Models events occurring at a constant average rate $\lambda$ in a fixed interval of space or time.

$$P(X=k) = \frac{e^{-\lambda} \lambda^k}{k!}$$

Cyber Example: Predicting the probability of receiving more than 500 packets per second on a network firewall during a DDoS attack.

3. Lognormal Distribution (Continuous)

Used for variables that are non-negative and highly skewed. Ideal for **Cyber Breach Financial Losses**.

Cyber Example: Breach costs multiply across layers (Recovery $\times$ Legal Fines $\times$ Customer Churn). Lognormal distribution captures the rare but critical "Mega-Breaches" in its fat tail.

10-Mark Blueprint: Hamming Code

Definition (2) + Step-by-Step Algorithm (4) + Full Numerical Example (4)

Hamming Code Algorithm

The Algorithm Step-by-Step

1

Redundant bits calculation: Find the number of parity bits required using the inequality: $$2^r \ge m + r + 1$$ where $m$ is the data bits length and $r$ is parity bits length.

2

Placement: Place the redundant parity bits at positions that are powers of 2 (i.e. indices 1, 2, 4, 8...). Fill remaining indices with data bits.

3

Calculation: Use Even or Odd Parity for covered positions to solve the parity bit values.

10-Mark Blueprint: AES Architecture

Intro (2) + Round Breakdown (4) + Detailed Four Operations (4)

AES: Advanced Encryption Standard

AES is a symmetric block cipher that processes data blocks of 128 bits. It utilizes a Substitution-Permutation Network (SPN) structure, structuring bytes in a $4 \times 4$ column-major matrix.

Rounds count

  • AES-128: 10 Rounds
  • AES-192: 12 Rounds
  • AES-256: 14 Rounds

The State Block

Data blocks of 16 bytes are arranged in a 4x4 matrix, enabling parallel byte operations.

Keys Schedule

Original key is expanded into separate subkeys, one for each round (plus initial key addition).

The 4 Transformation Functions

1

SubBytes (Confusion)

Non-linear byte substitution using a lookup table (S-Box) to achieve confusion.

2

ShiftRows (Diffusion)

Transposes bytes cyclically in rows 2, 3, and 4 to distribute columns influence.

3

MixColumns (Diffusion)

Linear matrix multiplication of state columns in GF(2^8) (omitted in final round).

4

AddRoundKey

Exclusive OR (XOR) operation applied to each column of the State matrix with the corresponding Round Key.

10-Mark Blueprint: Shamir's Scheme

Threshold Concept (2) + Polynomial Logic (3) + Simple Numerical (5)

Shamir's (k, n) Threshold Scheme

A threshold scheme splits a sensitive secret into $n$ pieces (shares) in such a way that combining any threshold of $k$ shares recovers the secret, while any subset of size $k-1$ or less leaks absolute zero information.

The Polynomial Method

Core Property: Interpolation logic states that it takes exactly $k$ coordinates/points to uniquely resolve a polynomial of degree $k-1$.

$$f(x) = S + a_1x + a_2x^2 + \dots + a_{k-1}x^{k-1}$$

Where the constant term S (where x=0) is the Secret.
